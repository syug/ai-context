## 重要な判断ログ

### cmux API の正しい使い方（ソース解析 3/5）
- `workspace_id` でフィルタ、`surface_id` UUID で送信、`workspace.select` は使わない

### tmux 全面移行の決定（3/5）
- cmux API の癖（3 RPC、\r ハック、UUIDオンザフライ解決、ソケット断続切れ）を排除
- `tmux send-keys` で確実送信
- sesh でセッション定義一元管理
- cmux は GUI、tmux はデータ転送 — 役割分離

### send-keys の Enter 問題（3/6）
- `tmux send-keys -t session -- 'text' Enter` だと `--` フラグが `Enter` キーの解釈を妨げる
- **修正:** テキスト送信と Enter を2コマンドに分離

### LaunchAgent の Mach Bootstrap 問題（3/3 → 3/6 解決）
- cmux ソケット依存時: LaunchAgent から接続不可（macOS セッション隔離）
- tmux send-keys 移行後: 問題解消（tmux はユーザーセッションで動作）
- PATH に `/opt/homebrew/bin` を含める必要あり（tmux コマンドの場所）

### Slackスレッド vs チャネル
- スレッド内 `$ws` は非対応（API制約）→ 仕様として受容
