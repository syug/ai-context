## 多層防御アーキテクチャ全体像

```
[フロントエンド]
  100文字制限 + T&C同意必須（PII disclaimer含む） + trim()
    ↓
[WAF] 6ルール, IP 2000req/5min
  ↓
[API Gateway] CORS whitelist, 30req/s throttle, Request Validator
  ↓
[Lambda Authorizer] Origin + Token
  ↓
[Andon Cord] USE_BEDROCK_API env var
  ↓
[validateInputText] 100文字切捨 + lowercase + マルチバイト置換 + 絵文字除去 + URIエンコード + CRLF除去
  ↓
[buildPrompt] <action>タグでラップ
  ↓
[Bedrock RetrieveAndGenerate] Knowledge Base + Guardrails
  ├─ Content Filters: 全カテゴリ HIGH
  ├─ Denied Topics: 28個
  ├─ Word Filters: 1,494語（多言語）
  └─ PII Filters: 全カテゴリ BLOCK
    ↓
[validateAnswer] YES/NO/BLOCKED 正規化（デフォルト NO）
  ↓
[DynamoDB] 全リクエスト/レスポンス ログ保存（KMS暗号化, PITR有効）
  ↓
[Response] 本番: {answer} のみ返却（LLM生テキスト非公開）
```

## フロントエンド側ガードレール

### 文字数制限（100文字）
**ファイル:** `src/component/ScreenAI/ScreenAI.tsx`
```tsx
const CHARACTER_LIMIT = 100;
const reachedCharactersLimit = userInput.length > CHARACTER_LIMIT;
const isFormValid = !reachedCharactersLimit && termsAccepted && userInput.length > 0;
```
- 101文字以上で無効化、WarningInCircleIcon + 「Character limit reached」表示
- submitボタンは `disabled={!isFormValid}` で制御

### T&C 同意チェックボックス（PII disclaimer含む）
```
You are responsible for any text you type into the AI Tool and you agree that
you will not use the AI Tool to enter, upload, share or create any content
that is harmful, abusive, illegal, misleading or violates the rights of
others. You agree that you have all necessary rights and permissions to enter
information into the AI Tool and that you will not enter sensitive personal
data of another individual.
```

```
A customer's prompts and the results generated by the AI Tool may be reviewed
through both automated and manual methods for abuse prevention and content
filtering purposes.
```

### AI目的 disclaimer
```tsx
*AI powered for entertainment purposes and assessment of what is a "Thing".
```

### API呼び出し前のプリプロセス
```tsx
const trimmedInput = arg.userInput.trim();
```
- `trim()` のみ。絵文字除去・PII検出等のクライアントサイド処理はなし（サーバー側に委任）

### フロントエンド側に存在しないガードレール
- 禁止文字フィルタ（絵文字・特殊文字）
- PII検出ロジック
- HTMLサニタイゼーション
- input の maxLength HTML属性

## バックエンド側ガードレール

### 1. validateInputText — 5段パイプライン
**ファイル:** `src/utils/validateInputText.ts`

```typescript
export const validateInputText = (str: string) =>
  ignoreCrlf(
    encodeURIComponent(
      stripEmoji(
        replaceMultiBytePunctuations(
          String(str ?? '')
            .substring(0, INPUT_TEXT_CHARACTER_LIMIT)  // 100文字
            .toLowerCase(),
        ),
      ),
    ),
  );
```

| ステップ | 処理 |
|---------|------|
| 1 | 100文字で切り捨て + toLowerCase() |
| 2 | マルチバイト引用符（U+201C, U+201D）を ASCII `"` に変換 |
| 3 | 絵文字除去（`node-emoji` の `strip()`） |
| 4 | URIエンコード（`encodeURIComponent()`） |
| 5 | CRLF除去（`\n`, `\r\n`, `\r` を空文字に） |

### 2. Andon Cord

if (process.env.USE_BEDROCK_API !== 'true') {

### 4. Bedrock Guardrails 設定
**ファイル:** `lib/constructs/bedrock-guardrails.ts`

#### Content Filters（全カテゴリ HIGH）
対象: SEXUAL, VIOLENCE, HATE, INSULTS, MISCONDUCT, PROMPT_ATTACK
- 全て Input: HIGH, Output: HIGH
- PROMPT_ATTACK のみ Output: NONE

#### Denied Topics（28個）
ビルトイン5個:
- financialAdviceTopic, politicalAdviceTopic, inappropriateContent, legalAdvice, medicalAdvice

カスタム23個（`lib/assets/guardrails/denied-topics.json`）:
- Alcohol, Brand safety, Celebrities, Children, Competitors, Consumption, Copyright, Death, Defame, Hate, Hatred implication, Law, Logo, Misinformation, Politics, Politics Mars, Prompt injection, Derogatory, Religion, Sexually explicit, Violence, Security, Extremist terms

#### Word Filters（1,494語）
**ファイル:** `lib/assets/guardrails/filtered-words.txt`
- 競合ブランド名、白人至上主義/極右用語、災害関連、コロナ関連、軍事/テロ関連、ゲーム関連、不適切表現
- 多言語（アラビア語、オランダ語、ポーランド語、フランス語、スペイン語、イタリア語等）

生成スクリプト: `scripts/generateGuardrailAssetsFromExcel.cmd.ts`
- Excelファイルから自動生成
- 3語超のフレーズは除外（Bedrock制約）
- Bedrock quota上限 10,000語をチェック

#### PII Filters（全カテゴリ BLOCK）
6カテゴリの全PII型を BLOCK:
- bedrock.General, bedrock.InformationTechnology, bedrock.Finance
- bedrock.USASpecific, bedrock.CanadaSpecific, bedrock.UKSpecific

カスタム正規表現:
```typescript
{ name: 'CUSTOMER_ID', pattern: '/^[A-Z]{2}d{6}$/', action: 'BLOCK' }
```

#### Blocked Messages
```typescript
blockedInputMessaging: '{ "answer": "BLOCKED", "comment": "Sorry, the model cannot answer this question." }',
blockedOutputsMessaging: '{ "answer": "BLOCKED", "comment": "Sorry, the model cannot answer this question." }',
```

### 5. validateAnswer — レスポンス正規化
```typescript
const validateAnswer = (answer: string): Answer => {
  const upperCased = answer?.toUpperCase();
  return upperCased === 'YES' || upperCased === 'NO' || upperCased === 'BLOCKED'
    ? upperCased
    : 'NO';  // デフォルトは NO
};
```
- YES/NO/BLOCKED 以外 → NOにフォールバック
- 本番では `answer` のみ返却、LLM生テキスト非公開

### 6. DynamoDB ログ保存

| AWSManagedRulesCommonRuleSet | Block |

### 8. API Gateway
- Lambda Authorizer: Origin whitelist（amazon.co.jp, amazon.com.au, console.harmony.a2z.com, localhost）+ Token
- Request Validator: BETA/PRODでは `additionalProperties: false`
- Throttling: 30 req/s, Burst 20
- エラーマスキング: 本番では常に 'Unexpected error'
