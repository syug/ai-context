## 背景 / タイムライン

1. **2025年**: Mars Dine "Cat Decoder" キャンペーン — "Ignored to Adored" ブランドプラットフォームを確立した成功事例。猫の行動を解読し、飼い主をDineの食事に誘導するインタラクティブツール。

2. **2026年企画（エージェンシーフェーズ）**: Mars Inc. x Publicisが **"Amazon Dine Video" / "pspspsuedoscience"** という初期コンセプトを策定（17スライドのPDFブリーフ）。エージェンシーはPrime Videoのプレミアムパートナーシップを提案し、猫を引き付ける音（pspsps音、ジングル、ゴロゴロ音、鳴き声）をPrime Videoコンテンツに代替オーディオトラックとして直接埋め込む — 本質的には言語選択のような「猫向けオーディオオプション」を提案。コンセプトに含まれる要素：
   - Amazon Prime内にキュレーションされた「猫向けジャンル」セクション（BTYBブランドロウ）
   - 猫の飼い主に猫向けオーディオへのオプトインを促すインタラクティブプレロール広告
   - 12日間コネクションチャレンジによるPR拡散
   - 拡張機能: Prime登録で12日間分のDine無料特典、猫が好む音を集めた限定Amazon Musicチャンネル
   - アンプリフィケーション: ファーストインプレッションテイクオーバー、特定タイトルへのスポンサーシップアライメント、QR対応「paws ads」

3. **エージェンシー案をAU BILが却下** — Luke Thistleton（AU BILクリエイティブリード）が、エージェンシー（PublicisのWarren & Leah）に以下のフィードバックを提供（2026-02-10 Slack DMにて共有）:

   **戦略的理由（主因）:** *「"cat-dubbed media"のアイデアは[Cat Decoderの]ロジックを本質的に逆転させている。飼い主と猫の問題を解決したり、製品の価値を示したりするのではなく、人間を一時的に楽しませつつ猫を刺激するだけのノベルティ仕掛けだ。ブランドは行動変容の原因ではなく、トリックのスポンサーになってしまう。一方のアイデアは食べ物が関係改善の理由になり、もう一方は広告がエンターテイメントで製品は付随的 — 注目は集めるが信頼は生まない。」*

   **技術的理由（副因）:** *「実行面について: これは広告（撮影されたTVC）としてしか実現できない。現在、プラットフォームにオーディオを直接組み込む能力はない。」*

4. **Cat Decoder再実施を検討** — チームは2025年のCat Decoderを2026年に再実施することを議論。

5. **新アイデアの策定** — 最終的に新しいコンセプトを作成する方針となり、"Ignored to Adored V3" で3つのアイデアが完成。

6. **2026-03-11**: AU BILクリエイティブチームがクライアントに提示。

7. **現在**: クライアントからのフィードバック待ち。

> **注記**: エージェンシーの "pspspsuedoscience" コンセプトはBILのIdea 03（"Dinner and a Show"）と多くの共通点を持つ — いずれもPrime Video視聴中に猫を引き付けるオーディオを中心としている。BIL版はPVコンテンツへのオーディオ埋め込みから、セカンドスクリーンコンパニオン体験へと方向転換しており、技術的により実現可能。

## Idea 01: Cat Love Keyboard — "Music to move your feline friend"

### ブリーフサマリー

90年代スロージャム・ラブバラードにインスパイアされたデジタルキーボード（Amazon.com/DineKeyboardでのモバイルWeb体験）で、飼い主が猫向けのパーソナライズされたセレナーデを作曲できる。David Teie（*Applied Animal Behaviour Science*）の種固有音楽研究に基づく。

### メカニクス（ブリーフより）

1. Amazon.com/DineKeyboard にアクセス → スマートフォンがCat Love Keyboardに変身
2. バッキングトラックを選択、猫が好む音（ゴロゴロ音ループ、鳥のチャープ音、Dine缶を開ける音）をレイヤー
3. 猫に向けてライブ演奏 → 保存 → Dineブランドストアにシェア
4. 楽曲シェアで12-Day Dine Meal Challengeの割引コードを獲得
5. シェアされた楽曲はAmazon Music上の**Dine Cat Love Playlist**に集約、Amazon Alexa経由でプロモーション

### 科学的根拠（ブリーフより）

David Teie研究に基づく — 猫の生物学に基づくリズム（ゴロゴロ振動、吸乳リズム、グルーミングパターン）、猫の発声にマッチするスライディングピッチ、猫の声域に合わせた300-3,000Hz周波数帯、馴染みのある生活音（ゴロゴロ、グルーミング、鳥のチャープ）。

### クリエイティブテリトリー

90年代ラブバラードへのノスタルジア。架空マスコット "Meow-ceal Bolton"（ロシアンブルー猫）の可能性。TikTok/Reels（音声ON環境）、インフルエンサーコンテンツ、Prime Videoメディア、Amazon DSP、カスタムブランドストアで展開。

### 総合フィージビリティ: 🟡 Yellow（中リスク）

コアのインタラクティブキーボード（Web Audio API + Brand Store）は **🟢 Green（低リスク）**。Amazon Music UGCパイプラインが **🔴 Red（高リスク）** のブロッカー。

### 主要リスク

| リスク | 深刻度 | 詳細 |
|------|----------|--------|
| Amazon Music UGCパイプライン | 🔴 Red（高リスク） | ユーザー生成オーディオをAmazon Musicプレイリストに取り込む仕組みが存在しない。既存事例（Vitaminwater, Cocktail Cabinet）はすべて事前キュレーションされたプレイリストのみ |
| iOS WebViewオーディオ | 🟡 Yellow（中リスク） | WKWebViewではAudioContext初期化にユーザージェスチャーが必要。UXデザイン（「タップして開始」）で解決可能 |
| クロスチーム依存関係 | 🟡 Yellow（中リスク） | BIL-E + Amazon Music + Alexa + Promotions — 各チームがタイムラインリスクを追加 |
| モバイルオーディオレイテンシ | 🟡 Yellow（中リスク） | ローエンドAndroid端末では50ms以上の遅延が発生する可能性。キーボードのタップ→音の操作には低レイテンシが必要 |
| ANZ Brand Store | 🟡 Yellow（中リスク） | 確認された先行事例はすべてUS/UK/EU/JP。amazon.com.auでのカスタムBrand Store体験は要確認 |

3. **カスタムSkillよりAlexa Theme** — Barbie先行事例（29.2万ダウンロード）、承認プロセスが軽量
4. **段階的ビルド** — Phase 1: キーボード + 割引コード（クロスチームブロッカーなし）。Phase 2: Amazon Musicキュレーションプレイリスト + Alexa Theme

## Idea 02: Adore Des Chats — "The fragrance felines can't ignore"

### ブリーフサマリー

猫の愛情を勝ち取るために設計された限定版フレグランス。12-Day Dine Feeding Challengeの完了で無料プレゼント。古典的条件付け（パブロフの原理）を活用 — 食事時やリラックス時にスプレーし、ポジティブな香りの連想を構築。Dine缶をモチーフにしたカスタムボトルにゴールドアトマイザースプレー。

### 主要フレグランスノート（ブリーフより）

1. **Silver Vine（Actinidine）:** 猫の約80%が反応（Catnip非反応猫も含む）。ヒト向けプロファイル: 穏やかなウッディ、お茶のような香り。
2. **Valerian Root（Actinidine & Valerenic）:** 猫を穏やかに刺激した後リラックスさせる。ヒト向け: 温かみのあるムスキー、ウッディ。
3. **Catnip（Nepetalactone）:** 猫の50-70%が強く反応。ヒト向け: ソフトミント。

### コンディショニングメカニクス（ブリーフより）

食事時＋ソファでのリラックス時にスプレー。時間の経過とともに、香りがDineフードのポジティブな期待と結びつき、その興奮が愛情あるスナグリングに転化される。

### クリエイティブテリトリー

高級フレグランス広告のトロープ（おどけたフランス語ナレーション）。フレグランス開発と猫テスティングの裏側コンテンツ。Prime Video、インフルエンサープログラム、PR、ブランドストア、Amazon DSPで展開。

### 総合フィージビリティ: 🟡 Yellow（中リスク）

デジタルコンポーネント（Brand Store, PV, DSP, Influencer）はすべて **🟢 Green（低リスク）**。物理フレグランス製品が **🔴 Red（高リスク）** のリスクベクター。

### 主要リスク

| リスク | 深刻度 | 詳細 |
|------|----------|--------|
| 製品安全性 | 🔴 Red（高リスク） | 猫はエッセンシャルオイルに対して非常に敏感。キャリア製剤（アルコール、固定剤、合成化合物）には獣医毒性学レビューが必要。Catnip/Silver Vine/Valerian自体は一般的に安全 |
| 規制（APVMA） | 🔴 Red（高リスク） | 動物の行動に影響を与える製品として販売される場合、獣医化学製品としての登録が必要となる可能性。登録期間: 6-12ヶ月 |
| 製造リードタイム | 🔴 Red（高リスク） | カスタムボトル（Dine缶風 + ゴールドアトマイザー）にはインダストリアルデザイン、金型作成、生産が必要。最低12-16週間 |
| フルフィルメント | 🟡 Yellow（中リスク） | BILは通常物理製品の発送を行わない。アルコールベースのフレグランスにはFBA危険物制限の可能性あり |
| チャレンジ完了トラッキング | 🟡 Yellow（中リスク） | 12日間の給餌行動を検証するネイティブAmazonメカニズムがない。購買ベースのプロキシが最もシンプル |
| TGAへの影響 | 🟡 Yellow（中リスク） | 治療的効能を謳う場合（例: 「猫を落ち着かせる」）、TGA分類が適用される可能性。パブロフ的フレーミングであれば回避可能 |

### コンポーネント評価

| コンポーネント | フィージビリティ | 備考 |
|-----------|------------|-------|
| Brand Storeカスタム体験 | 🟢 Green（低リスク） | 実績十分。チャレンジトラッカー、商品ショーケース、フレグランスストーリー |
| Prime Videoメディア統合 | 🟢 Green（低リスク） | 標準的なメディアバイ |
| コンテンツ制作（高級広告パロディ + BTS） | 🟢 Green（低リスク） | 標準的なBIL制作。フランス語VOパロディはクリエイティブ面であり、技術的に複雑ではない |
| Amazon DSP + Influencer Program | 🟢 Green（低リスク） | 標準的なアクティベーションチャネル |

| リワード引き換えフロー | 🟡 Yellow（中リスク） | 選択肢: プロモコード→ASIN、3PL経由の外部フルフィルメント、限定数の懸賞 |

| フルフィルメント/物流 | 🟡 Yellow（中リスク） | ガラスボトル = 破損リスク + 危険物（アルコール含有） |

### 推奨ピボット

1. **限定PR/インフルエンサーシーディング（50-200個）** — 一般チャレンジ完了者にはデジタルリワードを提供
2. **ブランド（Purina/Nestle）が製造を担当** — BILの役割: クリエイティブコンセプト + メディア + Brand Store。Nespressoモデルを踏襲
3. **購買トリガー型チャレンジ** — 「Dine 12パック購入 = チャレンジ完了」。購買データで検証可能
4. **物理配布は懸賞方式** — 全完了者が限定版フレグランスの抽選に参加
5. **ピッチ段階で規制評価を開始** — APVMAのタイムラインが遅れるとキャンペーン全体が頓挫する可能性

### 規制上の制約

- **APVMA**: 動物の行動に影響を与える製品は登録が必要な可能性（6-12ヶ月）
- **AICIS**: オーストラリア市場向けの化粧品/フレグランス成分コンプライアンス
- **ACCC**: 「猫の愛情を勝ち取る」というマーケティング表現には根拠の裏付けが必要
- **動物福祉**: 「猫でのテスト」を含むBTSコンテンツは動物福祉基準に準拠する必要あり

## Idea 03: Dinner and a Show — "Make a date to settle in and snuggle"

### ブリーフサマリー

初期の "PSPSPS" / エージェンシーコンセプトの進化版。よりコンシューマーレディに再構築。猫にDineを与えた後、飼い主がPrime Videoを視聴しながら、スマートフォンのセカンドスクリーン体験が猫専用に設計されたジェントルなオーディオバスを再生 — 猫のリラックスとスナグリングを促進。

### メカニクス（ブリーフより）

1. 猫にDineを与え、Prime Videoの視聴準備
2. スマートフォンでDine "Dinner and a Show" セカンドスクリーン体験を起動、近くに配置
3. スマートフォンが猫向けオーディオを再生（ブリーフでは人間の可聴域を超える20kHz+、猫向けに最大65kHzと記載 — **下記の重要評価を参照**）
4. アプリが番組に合わせて音量/サウンドスケープを動的に調整
5. 長時間利用でDine製品の小額割引をアンロック

### 科学的主張（ブリーフより）

猫は最大65kHzまで聴取可能（人間は最大20kHz）。高周波音は飼い主に聞こえないまま猫の注意を引き、中断のない共有の時間を実現する。**注意: この主張は技術的に問題がある — 下記の高周波オーディオ評価を参照。**

### クリエイティブテリトリー

「ディナーとショー」を普遍的なデートの儀式として、猫と飼い主の関係にリフレーム。コンテンツの切り口: 気まずいファーストデート、壮大なロマンティックジェスチャー、猫の「恋愛エキスパート」。インフルエンサーとの自然な統合（猫とのデートナイト）、Prime Videoロマンティック番組との連携。

### 総合フィージビリティ: 🟡🔴 Yellow/Red（ピボットにより🟡 Yellowに改善可能）

セカンドスクリーンインフラは強い先行事例があり **🟢 Green（低リスク）**。超音波オーディオの前提（20kHz以上）は **消費者向けスマートフォンでは物理的に不可能**。

### 重要: 高周波オーディオ評価

**ブリーフでは「人間の可聴範囲を超える20kHz以上、猫向けに最大65kHz」の音をスマートフォンで再生すると記載されている。これは消費者向けハードウェアでは物理的に不可能。**

| デバイス | スペック上の上限 | 実用的な上限 |
|--------|-------------------|----------------------|
| スマートフォンスピーカー | ~20kHz | 12-16kHz |
| テレビ内蔵スピーカー | ~20kHz | 12-15kHz |
| Amazon Echo（ツイーター付き） | ~20kHz | 18-20kHz |
| Echo Dot | ~20kHz | 15-18kHz |
| ブックシェルフスピーカー | ~20-22kHz | 20kHz |

**不可能な理由:**
- スマートフォンスピーカードライバー（7-15mm）は、意味のあるSPLで20kHz以上の振動ができない
- 44.1/48kHzの標準DACではNyquist最大値が理論上~22-24kHzだが、スピーカーはそれすら再生できない
- アンプ設計は100Hz-16kHzに最適化されており、超音波コンテンツを能動的にフィルタリング
- 65kHzには128kHz以上のサンプリングレートと専用トランスデューサーが必要 — 消費者向けデバイスには存在しない

**実現可能な範囲:**
- **8-18kHz帯域**: 猫はよく聞き取れるが、30歳以上の人間には聞き取りにくい（加齢性難聴）
- **猫が敏感な帯域（~2-8kHz）**: 人間にも聞こえるが、心地よいアンビエントサウンドとしてデザイン可能
- **種固有の音楽パターン**（Teie/Snowdon研究）: リラックス効果が確認されており、人間にも完全に聞こえる

### オーディオフィンガープリンティングの制約

オーディオフィンガープリンティング（Shazam方式）は、スマホの**マイク**でTV音声をキャプチャしてデータベースとマッチングする。Idea 03では、同じスマホが同時に**スピーカーから猫オーディオを再生**している:

- スピーカー出力がマイクにフィードバック（音響結合）
- TV音声信号がマスクされ、フィンガープリントのSNR（信号対雑音比）が劣化
- マッチング精度が大幅低下、または完全失敗

回避策（エコーキャンセレーション、デバイス分離）はBrand Store Webアプリとしては過度に複雑。**同じデバイスで聴取と再生を同時に行う場合、オーディオフィンガープリンティングは実質不可能。**

### 動的調整: PV同期複雑性の根本原因

ブリーフには「*アプリが番組に合わせて音量/サウンドスケープを動的に調整*」と記載されているが、なぜこれが必要かの説明はない。

**意図（推定）:**
- TVが大音量（アクションシーン）→ 猫がTVノイズに負けないよう猫オーディオを**上げる**
- TVが静か → 猫オーディオは変更不要だが、人間の快適さのために少し下げてもよい

ブリーフには「番組に合わせる」方法の詳細が記載されていない。方法によって技術的複雑性は大きく異なる:

| 「番組に合わせる」方法 | 必要な技術 | 複雑性 | PV同期の要否 |
|----------------------|-----------|--------|-------------|
| **リアルタイムシーンレベル同期**（再生位置に基づきシーンごとにオーディオ調整） | STS API / オーディオフィンガープリンティング / 手動タイマー同期 | 🔴 Red | 必要 — どのシーンが再生中か正確に把握する必要あり |
| **リアルタイムTV音量トラッキング**（マイクでTV音量変化を検知、猫オーディオを調整） | マイク入力 + 音量検知 | 🟡 Yellow — ただし猫オーディオ同時再生時のスピーカー/マイク競合で精度低下 | 不要だがマイクアクセスが必要 |
| **ジャンル/ムードプリセット**（ユーザーが「アクション映画」「ロマコメ」「ドキュメンタリー」等を選択、対応プリセットの猫サウンドスケープを再生） | ユーザー選択 → プリセットサウンドスケープ。PV連携なし | 🟢 Green — 極めてシンプル | 不要 |
| **調整なし**（TVコンテンツと無関係に一定のリラックス猫オーディオを再生） | Web Audio APIで一定のオーディオを再生 | 🟢 Green — 最もシンプル | 不要 |

**問題:** このメカニクス1つが、PV同期の全技術的複雑性の根本原因。リアルタイムシーンレベル同期（1番目の方法）のみがPV連携を必要とし、かつ最も複雑。残りの3つの方法はPV同期を一切必要とせず、下2つはBrand Store上で容易に実装可能。

**推奨:** Chris Wilsonにどの方法を想定しているか確認する。ジャンルプリセットまたは一定オーディオで十分であれば、Idea 03の技術リスクはYellow/RedからGreenに低下する。

**猫の行動科学的観点:** 猫は動的に変化するサウンドスケープよりも、**一貫した予測可能な穏やかなオーディオ**からより大きなリラックス効果を得る。突然の音量変化やトーンの変化は猫を警戒させ、リラックス目的に逆行する可能性がある。ブリーフ自体が引用する科学的根拠（種固有音楽研究）も、安定したリズミカルなパターンを支持しており、TV連動型の反応的オーディオではない。

**結論:** 「動的調整」を外すと、Idea 03は劇的にシンプルになる:
- PV同期不要 → STS APIアラインメントも不要
- フィンガープリンティング不要 → マイク/スピーカー競合なし
- スマホは一定のリラックスサウンドスケープを再生するだけ → Brand Store上のWeb Audio API（実証済み、🟢）
- "Dinner and a Show" のクリエイティブフレーミングは依然として成立: 猫にDineを与える → PVをつける → 猫オーディオを開始 → 一緒に楽しむ
- PV連携は純粋に**メディア/広告の関係**（標準的な広告出稿）に留まり、技術的同期の課題ではなくなる

### 推奨ピボット

1. **オーディオコンセプトのリフレーム（必須）**: 「超音波・猫専用オーディオ」を「猫に最適化された科学的設計のリラックスサウンドスケープ」に変更。8-18kHz帯域は実用的であり、成人にはあまり聞こえない
2. **動的PV同期の廃止（強く推奨）**: TVコンテンツと無関係に一定のリラックスオーディオを再生する方式に変更。最大の技術リスクを完全に排除しつつ、猫のリラックスとしても科学的により正しい
3. **PV同期を維持する場合**: SSE BetaプログラムにSTS API経由でアライン。手動エピソード選択+タイマーが最もシンプルなフォールバック。オーディオフィンガープリンティングはスピーカー/マイク競合により不可
4. **Coca-Cola WebSocket/MessageRoomスタックを再利用**: PV同期を維持する場合のみ必要。本番環境対応済みアーキテクチャ、引き継ぎはjtransuに連絡
5. **Brand Store Webアプリ**: App Store承認不要、DineパッケージのQRコードでアクセス
6. **Echoを再生デバイスとして検討**: スピーカー品質が優秀（18-20kHz）、Alexaエコシステム統合、自然なフィット。PV同期を追求する場合、Echoで猫オーディオ再生 + スマホでフィンガープリンティングとデバイス分離も可能
