## 重要な判断ログ

### なぜAISmith流用を選択したか
- Enterprise版はEngine組織限定、PATは管理者ブロック、AISmith方式が唯一のPAT不要パス

### Asana通知のハイブリッド方針
- Asana Workflowsにコメントトリガーが存在しない → Webhook併用が必須

### Slack MCP直接投稿 vs Webhook
- Slack MCP `post_message` は自分名義 → 通知来ない。Webhook → bot名義 → 通知届く

### LaunchAgentからcmux接続不可
- macOS Mach Bootstrapセッション隔離。cmuxワークスペース内で実行で解決

### cmuxソケット直接接続の採用理由
- cmux CLIはインタラクティブ用途。ソケットAPIがプログラマティック利用の正しいインターフェース

### ブリッジ常駐化: LaunchAgent → cmuxワークスペース
- CMUX_WORKSPACE_ID + fish config で自動起動。start.sh while trueで自動復帰

### SSH/Termius 正式不要判定
- Director Mode思想との不一致。Self-reporting protocolで代替

### Bridge双方向化の暗黙的棄却
- Webhookが CC→User 即時通知を十分カバー。双方向化はROI低い
