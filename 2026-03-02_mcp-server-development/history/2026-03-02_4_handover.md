## 重要な判断ログ

### Asana通知の採用
- Slack/Asana MCPは自分のトークンで投稿 → 自分名義 → 通知来ない
- Asana→Slack Workflow連携でAsana bot名義に変換 → 通知来る
- これが唯一の「追加インフラなし」で通知を実現する方法

### Slack→cmuxブリッジの設計
- `$` プレフィックス採用（`@` はSlackメンション、`>` はブロック引用でエスケープされる）
- MCP SDK `StdioClientTransport` でSlack MCPサーバーに接続（`aim` ラッパー経由必須）
- cmuxは独自ソケットベース（tmuxではない）、`cmux send` / `cmux send-key` で入力送信
- ワークスペース番号・名前のあいまいマッチ対応

### OpenClaw BAN
- 2026-02-04 CorpSec決定。セキュリティ脆弱性 + Meta inbox wipe事件
- 代替: HaroldCLI（RustHack 2026、Slack Socket Mode、OPUS承認必要）

### Bedrock経由の制約
- /rc 不可、クラウド版不可 → ローカルMac縛り
- SSH + Asana通知 + Slackブリッジの組み合わせが最適解
