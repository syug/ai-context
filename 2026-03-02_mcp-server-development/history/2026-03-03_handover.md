## 重要な判断ログ

### なぜAISmith流用を選択したか
- Enterprise版はEngine組織限定、PATは管理者ブロック、AISmith方式が唯一のPAT不要パス

### Sub-agent権限問題
- OneDrive/Google Drive配下でagent書き込みブロック → メインスレッドで対処

### Amazon CA証明書の扱い
- 3段階フォールバック、実機テストでシステムデフォルトで十分

### PAT申請（バックアップ・不要になった）
- SSO方式が正常動作したため不要

### Asana通知の採用
- Asana→Slack Workflow連携でbot名義変換が唯一の追加インフラなし通知方法

### Slack→cmuxブリッジの設計
- `$` プレフィックス、MCP SDK StdioClientTransport、cmux独自ソケット

### OpenClaw BAN / Bedrock経由の制約
- CorpSec BAN、ローカルMac縛り → SSH + Asana + Slackブリッジが最適解
